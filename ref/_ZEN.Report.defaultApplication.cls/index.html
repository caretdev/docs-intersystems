%ZEN.Report.defaultApplication

Class %ZEN.Report.defaultApplication Extends %RegisteredObject [ Deprecated, System = 4 ]

Parameters

SUPPORTMACROS

Parameter SUPPORTMACROS = 0;

If SUPPORTMACROS is 1 we canocalize the ZEN Report Definition and Display replacing all single-quoted attributes by double-quoted attributes Also we allow macros through the macrodef element

SQLCACHE

Parameter SQLCACHE As STRING = 1;

REPORTXMLNAMESPACE

Parameter REPORTXMLNAMESPACE As STRING;

If provided, this is the XML namspace that will be used for the generated XML report.

REPORTXMLNAMESPACEPREFIX

Parameter REPORTXMLNAMESPACEPREFIX As STRING;

If provided, this is the XML namspace prefix that will be used for the generated XML report.

NAMESPACEDECLARATIONS

Parameter NAMESPACEDECLARATIONS As STRING;

If provided, this is a string of XML namespace declarations.
For example, "xmlns='http://mydefault' xmlns:ns1='http://namespace1'".
The namespace declarations will go on the root element of the generated XML and also on the stylesheet element of the generated XSL.

EMBEDXSL

Parameter EMBEDXSL As BOOLEAN = 0;

Specifies whether XSLT is embedded in XML, if it is we must use explicit namespaces for generated XML

AGGREGATETAG

Parameter AGGREGATETAG As String;

REMOVENULLS

Parameter REMOVENULLS As BOOLEAN = 0;

If 1 (set) elements or attributes that contain only NULL are not generated The default is 0 to preserve past ZEN Report identification of null with empty string, so as not to break existing reports

Parameter HANG As Integer = 0;

then we return an error from rendering
when this parameter is "" the default or -1, then we can potentially wait forever to connect
The units are seconds.

if we do not render through the render server in the amount of time specified in this parameter
then we return an error from rendering
when this parameter is "" the default or -1, then we can potentially wait forever to render
The units are seconds.

SPLITANDMERGE

Parameter SPLITANDMERGE As BOOLEAN;

If SPLITANDMERGE set, along with REPEATINGELEMENT and COUNTREPEATINGELEMENT then the XML file is split on the repeating element called REPEATINGELEMENT every COUNTREPEATINGELEMENT elements. REPEATINGELEMENT is an element right under root element. Let ... be the repeating element. Then to the renderer the XML file looks like (...)*, 0 or more repetitions of the element .... The XML file can contain other sibling elements to as long as they are NOT mentioned in the Report Display.
